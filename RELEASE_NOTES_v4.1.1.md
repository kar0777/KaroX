# KaroX v4.1.1 — Надёжные обновления + чистка инструментов

## Критический фикс цепочки обновлений

- `bootstrap.ps1` / `bootstrap.sh` больше не содержат захардкоженный релиз (был `v3.16.2`): из-за этого `karox update` на любой старой версии переустанавливал 3.16.2 вместо последней версии.
- Bootstrap теперь сам резолвит целевой тег: параметр `-Branch` / переменная `KAROX_BOOTSTRAP_REF` → `RELEASE.json` на `main` (`tag`, затем `version`) → фолбэк `main`.
- `karox update` дополнительно передаёт целевой тег инсталлеру через `KAROX_BOOTSTRAP_REF` — установленная версия всегда совпадает с той, которую подтвердил пользователь.
- Диагностика без установки: `bootstrap.ps1 -ResolveOnly` / `bootstrap.sh --resolve-only` печатают резолвленный тег и выходят.

## Дедупликация MCP-инструментов (45 → 42)

- `karox_exec` влит в `karox_run`: новые параметры `argv_json`, `shell`, `cwd`, `env_json`, `stdin` — точный argv без shell-обёртки.
- `karox_checks_v2` влит в `karox_run_checks`: новый параметр `checks_json` — матрица с `allowFailure`/`retries` и структурным разбором ошибок (`firstError`).
- `karox_search_v2` влит в `karox_search`: новые параметры `regex`, `names_only`, `extensions`, `context`, `max_size`.
- Возможности не потеряны — удалены только дублирующие имена инструментов.

## Обновление

```
karox update
```

Или повторный запуск установочной команды из README — инсталлер идемпотентен и чинит битую установку.
