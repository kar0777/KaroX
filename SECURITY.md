# Безопасность

Star For KaroX открывает выбранный локальный репозиторий через HTTP API. Это удобно для AI-агентов, но такой доступ нужно воспринимать как рабочий инструмент с понятными границами, а не как абсолютную песочницу.

## Что защищено по умолчанию

Star For KaroX блокирует:

- `.env` и похожие env-файлы;
- SSH-ключи;
- token/cookie-файлы;
- файлы приватных ключей с расширениями `.pem`, `.key`, `.p12`, `.pfx`;
- выход за пределы выбранного репозитория;
- `git push`;
- изменение Git remotes;
- `ssh`, `scp`, `sftp`;
- опасные системные команды вроде `shutdown`, `diskpart`, `format`, `bcdedit`;
- прямые `git add` и `git commit` в режиме `Автопилот`.

Commit в автопилоте проходит через безопасный endpoint `/git/commit`: он проверяет ветку, список файлов, staged diff и базовое наличие секретов.

## Что не может быть полностью защищено

Если в самом репозитории есть опасные скрипты в `package.json`, Gradle, Makefile или других build-файлах, запуск тестов/сборки может выполнить эти скрипты.

Перед режимами `Автопилот` и `Полный режим` просматривайте неизвестные репозитории хотя бы в режиме `Безопасный просмотр`.

## Рекомендации

- Для неизвестных проектов начинайте с `Безопасного просмотра`.
- Для обычной разработки используйте `Автопилот`.
- `Полный режим` включайте только когда понимаете риски конкретного проекта.
- Не используйте Star For KaroX с репозиториями, где лежат реальные секреты.
- Не отправляйте `X-API-Key` обычным сообщением в чат. Вставляйте его только в защищённое поле интеграции.
- После завершения работы закрывайте окна сервера и туннеля или выбирайте пункт `Остановить серверы` в лаунчере.
