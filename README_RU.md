KaroX запускает ограниченный репозиторием OpenAPI-сервер, открывает его через защищённый туннель и формирует готовый handoff для AI. Вы выбираете проект и профиль доступа, а AI получает точную ветку, разрешения, live-preflight и правила безопасности.

**Почему KaroX**

- **Local-first:** исходный код остаётся на вашем компьютере, сервер ограничен выбранным репозиторием.
- **Явный контроль:** профили Observe, Build, Resume и Advanced показывают реальные разрешения.
- **Сначала контекст:** Mission Control формирует свежий безопасный бриф без секретов.
- **Безопасный Git workflow:** проверка ветки, cleanup generated-файлов, контролируемый commit и запрет push.
- **Для ежедневной работы:** двуязычный Flight Deck, история сессий, диагностика и установка одной командой.

## Первый запуск

```bash
karox
```

При первом запуске KaroX спрашивает:

1. **Язык** — Русский или English. Выбор сохраняется; позже его можно изменить через `G → L`.
2. **AI-клиент** — рекомендуется PromptQL; также доступны универсальный OpenAPI-режим и совместимость с letaido.

Для CI и запуска с перенаправленным вводом задайте язык заранее: `KAROX_LANGUAGE=en` или `KAROX_LANGUAGE=ru`. При обычном интерактивном запуске вопрос остаётся обязательным.

После заставки Star For KaroX автоматически открывается **KAROX / PROJECT FLIGHT DECK**. Анимацию можно отключить через `KAROX_NO_ANIMATION=1`; при перенаправленном выводе она отключается автоматически.

## Создание рабочего пространства

1. Нажмите `N`.
2. Выберите Git-репозиторий или вставьте полный путь.
3. Выберите профиль:
   - **Observe** — анализ только для чтения;
   - **Build** — отдельная ветка `promptql/*`, правки, проверки и безопасный commit;
   - **Resume** — продолжение текущей рабочей ветки;
   - **Advanced** — расширенные команды в репозитории.
4. Укажите метку сессии для истории. Она никогда не считается задачей для AI.
5. Дождитесь `● LIVE`.

KaroX сам запускает локальный API и Cloudflare Tunnel либо Tailscale Funnel. URL, ветку и разрешения вручную собирать не нужно.

## Передача сессии AI

Откройте карточку сессии:

| Клавиша | Действие |
|---|---|
| `V` | Локально проверить готовность для AI |
| `M` | Открыть контекст Mission Control |
| `C` | Скопировать connection prompt на выбранном языке |
| `T` | Скопировать шаблон реального ТЗ |
| `K` | Отдельно скопировать `X-API-Key` |
| `P` | Скопировать provider ID |
| `A` | Скопировать полный handoff |
| `L` | Показать логи |
| `S` | Остановить сессию |

**Перед handoff нажмите `V`.** KaroX локально вызывает `/session`, `/health`, `/git/status` и `/context/brief`, затем сверяет `repoRoot` и ветку с карточкой сессии.

Нажмите **`M`**, чтобы открыть **Mission Control**: актуальный безопасный бриф без секретного ключа со статусом задачи, разрешениями, чистотой Git, обнаруженным контекстом проекта, предупреждениями и рекомендуемым следующим шагом. AI может повторно вызывать `GET /context/brief`, не полагаясь на устаревший handoff. Предупреждение само ничего не меняет — существующий diff нужно просмотреть до продолжения.

Вставьте connection prompt в AI-клиент. Ключ вводите только в защищённую форму credentials — никогда не отправляйте его в чат. После успешного preflight пришлите отдельное сообщение с реальным ТЗ.

## Установка одной командой

### Windows

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -Command "irm https://raw.githubusercontent.com/kar0777/KaroX/main/bootstrap.ps1 | iex"
```

### macOS / Linux

```bash
curl -fsSL https://raw.githubusercontent.com/kar0777/KaroX/main/bootstrap.sh | bash
```

Основная команда — `karox`. `repopilot` сохранён как compatibility alias для старых установок.

## Терминальный интерфейс

KaroX использует единую адаптивную дизайн-систему в PowerShell и POSIX shell:

- сетка шириной 52–88 колонок остаётся читаемой в узком терминале;
- карточки workspace сразу показывают репозиторий, статус, ветку, доступ, туннель и AI-клиент;
- progress, success, warning, error и empty states оформлены единообразно;
- command bar и последовательность передачи AI рассчитаны на управление с клавиатуры;
- Mission Control показывает понятные действия вместо внутренних machine-кодов;
- структура, символы и подписи сохраняют смысл даже без цветного вывода.

## Навигация Flight Deck

| Клавиша | Действие |
|---|---|
| `N` | Новая сессия |
| номер | Открыть workspace |
| `G` | Настройки языка, AI-клиента и туннеля |
| `D` | Диагностика |
| `R` | Обновить |
| `U` | Очистить остановленную историю |
| `X` | Остановить все сессии |
| `Q` | Закрыть менеджер, не останавливая LIVE-сессии |

## Контракт безопасности

- каждый endpoint требует уникальный для сессии `X-API-Key`;
- доступ ограничен выбранным `repoRoot`;
- секретные пути и выход за пределы репозитория блокируются;
- writable-профили используют явную ветку и набор разрешений;
- большой вывод сохраняется через `capture=file`;
- generated-файлы очищаются через `/git/cleanup-generated`;
- commit выполняется только через `/git/commit`;
- workflow KaroX никогда не выполняет `git push`.

HTTP endpoints и security contract остаются обратно совместимыми.
