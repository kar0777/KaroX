# ★ KaroX v4.1.3 — Tailscale без тупиков и параллельные сессии

## Главное

- **Клавиша `T` теперь действительно открывает Tailscale.** На Windows KaroX запускает desktop-приложение, выполняет `tailscale up`, распознаёт URL авторизации и открывает страницу входа в браузере. На macOS/Linux страница входа также открывается автоматически, если CLI её вернул.
- **Провайдер туннеля выбирается отдельно для каждой новой сессии.** Глобальная настройка стала понятным значением по умолчанию, а перед запуском можно выбрать Cloudflare или Tailscale без возврата в настройки.
- **Несколько сессий больше не конфликтуют на Tailscale `:443`.** KaroX обнаруживает уже активную Tailscale Funnel-сессию, объясняет ограничение одного HTTPS listener на устройство и предлагает запустить дополнительную сессию через Cloudflare Tunnel.
- Исправлены устаревшие подсказки: интерфейс и ошибки теперь указывают `T`, а не `L`.
- Windows PowerShell и POSIX launchers ведут себя одинаково; добавлены отдельные regression-тесты и CI-проверка.

## Проверено

- Windows PowerShell 5.1 parser
- Bash syntax
- Tunnel/session regression suite
- KaroX unit tests
- Product doctor tests
- Tailscale readiness tests

## Обновление

```powershell
karox update
```

После обновления создайте новую сессию. Для первой сессии можно выбрать Tailscale; для второй и последующих выбирайте Cloudflare либо соглашайтесь на автоматическое переключение, когда KaroX обнаружит занятый Funnel.
